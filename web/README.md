# 🌐 Code-To-Prompt Web 界面

这是 Code-To-Prompt 的 Web 前端界面，让你可以通过浏览器轻松使用这个工具。

## ✨ 功能特性

- 📁 **拖拽上传** - 直接拖拽文件夹到浏览器
- 🎯 **文件过滤** - 按文件类型过滤（如 ts,tsx,js,jsx）
- 🔢 **Token 计数** - 实时显示 token 数量
- 📋 **一键复制** - 复制到剪贴板
- 💾 **下载文件** - 保存为 .txt 文件
- ⚡ **实时预览** - 查看生成的输出
- 🚨 **智能警告** - Token 数量过大时自动提示

## 🚀 快速开始

### 方法 1: 直接打开（推荐）

```bash
# 直接在浏览器中打开
start web/index.html

# 或双击 index.html 文件
```

### 方法 2: 使用本地服务器

```bash
# 使用 Python
cd web
python -m http.server 8000

# 使用 Node.js
cd web
npx http-server

# 使用 VS Code Live Server
# 右键 index.html -> Open with Live Server
```

然后访问 `http://localhost:8000`

## 📖 使用说明

### 步骤 1: 选择文件夹

- **方式 A**: 拖拽文件夹到上传区域
- **方式 B**: 点击"选择文件夹"按钮

### 步骤 2: 配置选项（可选）

- **文件类型过滤**: 输入文件扩展名，如 `ts,tsx,js,jsx`
- **Token 限制**: 设置最大 token 数，如 `50000`
- **尊重 .gitignore**: 自动排除 node_modules 等目录

### 步骤 3: 查看结果

工具会显示：
- 📊 文件数量
- 🔢 Token 数量
- 📝 字符数量
- ⚠️ 智能警告（如果需要）

### 步骤 4: 使用输出

- **复制到剪贴板**: 点击"复制到剪贴板"按钮
- **下载文件**: 点击"下载为文件"按钮
- **粘贴给 AI**: 打开 ChatGPT 或 Claude，粘贴内容

## 🎯 使用场景

### 场景 1: 代码审查

1. 拖拽项目文件夹
2. 设置文件类型过滤（如 `ts,tsx`）
3. 复制输出
4. 粘贴给 AI: "请审查这段代码"

### 场景 2: Bug 修复

1. 只选择相关模块的文件夹
2. 复制输出
3. 粘贴给 AI: "这个模块有个 bug: [描述]"

### 场景 3: 功能开发

1. 选择要修改的功能模块
2. 复制输出
3. 粘贴给 AI: "请帮我添加 [功能描述]"

## 💡 使用技巧

### ✅ 推荐做法

1. **使用文件类型过滤** - 减少不必要的文件
   ```
   ts,tsx,js,jsx  # 只看代码文件
   css,scss       # 只看样式文件
   ```

2. **设置 Token 限制** - 避免生成过大的输出
   ```
   50000  # 适合大多数场景
   ```

3. **分模块处理** - 不要一次性上传整个项目
   - ✅ 上传 `src/auth` 目录
   - ❌ 上传整个项目根目录

### ❌ 避免做法

1. ❌ 上传包含 node_modules 的目录
2. ❌ 上传整个大型项目（> 100k tokens）
3. ❌ 包含构建输出（dist/build）

## 🔧 自动排除的文件

Web 界面会自动排除以下文件和目录：

- `node_modules/`
- `dist/`, `build/`
- `.git/`, `.next/`, `.cache/`
- `coverage/`
- `*.log`
- `.DS_Store`
- `.env`, `.env.local`
- `package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`

## 📊 Token 数量参考

| Token 数 | 适用模型 | 建议 |
|---------|---------|------|
| < 4k | GPT-3.5 | ✅ 完美 |
| 4k-8k | GPT-4 | ✅ 很好 |
| 8k-32k | GPT-4-32k | ✅ 可以 |
| 32k-128k | GPT-4-turbo | ✅ 可以 |
| 128k-200k | Claude 3 | ⚠️ 接近上限 |
| > 200k | 无 | ❌ 必须分批 |

## 🚨 常见问题

### Q: 为什么 Token 数量这么大？

A: 可能包含了不必要的文件。解决方案：
1. 使用文件类型过滤
2. 只上传特定的子目录
3. 确保排除了 node_modules

### Q: 可以上传多个文件夹吗？

A: 目前只支持单个文件夹。如果需要多个模块，可以：
1. 分别上传每个模块
2. 手动合并输出

### Q: 支持哪些浏览器？

A: 支持所有现代浏览器：
- ✅ Chrome/Edge (推荐)
- ✅ Firefox
- ✅ Safari
- ⚠️ IE 不支持

### Q: 数据安全吗？

A: 完全安全！
- ✅ 所有处理都在浏览器本地完成
- ✅ 不会上传任何数据到服务器
- ✅ 不会保存任何文件
