## 🎨 界面功能

### 1. 上传区域

**功能：**
- 拖拽文件夹上传
- 点击选择文件夹
- 自动识别文件结构

**支持：**
- ✅ 单个文件夹
- ✅ 嵌套目录
- ✅ 大量文件（自动过滤）

### 2. 配置选项

**文件类型过滤：**
```
ts,tsx,js,jsx  # 只看 TypeScript 和 JavaScript
css,scss       # 只看样式文件
html,vue       # 只看模板文件
```

**Token 限制：**
```
50000   # 适合大多数 AI 模型
100000  # 适合 Claude 3
```

**尊重 .gitignore：**
- ✅ 自动排除 node_modules
- ✅ 自动排除 dist/build
- ✅ 自动排除日志文件

### 3. 统计信息

**实时显示：**
- 📊 文件数量
- 🔢 Token 数量（估算）
- 📝 字符数量

**智能警告：**
- 🚨 > 200k tokens: 严重警告
- ⚠️ > 100k tokens: 警告
- 💡 > 50k tokens: 提示
- ✅ < 50k tokens: 正常

### 4. 操作按钮

**复制到剪贴板：**
- 一键复制全部内容
- 自动显示成功提示

**下载为文件：**
- 保存为 `code-context.txt`
- 可以离线使用

**重新选择：**
- 清空当前结果
- 重新开始

### 5. 预览区域

**功能：**
- 显示前 1000 字符
- 查看输出格式
- 确认内容正确

## 💡 高级技巧

### 技巧 1: 分批处理大项目

```
1. 先拖拽 src/auth 目录
2. 生成并保存为 auth.txt
3. 再拖拽 src/users 目录
4. 生成并保存为 users.txt
5. 根据需要使用不同的文件
```

### 技巧 2: 精确控制 Token 数

```
1. 设置 Token 限制: 50000
2. 如果超过，会自动提示
3. 调整文件类型过滤
4. 或选择更小的目录
```

### 技巧 3: 快速测试

```
1. 准备一个小型项目文件夹
2. 拖拽到浏览器
3. 预期结果: 几千 tokens
4. 熟悉界面操作
5. 然后处理真实项目
```

## 🔧 技术实现

### 前端技术

- **纯 HTML/CSS/JavaScript** - 无框架依赖
- **File System Access API** - 读取文件夹
- **Clipboard API** - 复制到剪贴板
- **Drag and Drop API** - 拖拽上传

### 安全性

- ✅ **本地处理** - 所有操作在浏览器完成
- ✅ **无服务器** - 不上传任何数据
- ✅ **无存储** - 不保存任何文件
- ✅ **隐私保护** - 代码不离开你的电脑

### 兼容性

| 浏览器 | 支持 | 说明 |
|--------|------|------|
| Chrome | ✅ | 完全支持 |
| Edge | ✅ | 完全支持 |
| Firefox | ✅ | 完全支持 |
| Safari | ✅ | 完全支持 |
| IE | ❌ | 不支持 |

## 📊 性能

### 处理速度

| 文件数 | Token 数 | 处理时间 |
|--------|---------|---------|
| < 10 | < 5k | < 1 秒 |
| 10-50 | 5k-20k | 1-3 秒 |
| 50-100 | 20k-50k | 3-5 秒 |
| > 100 | > 50k | 5-10 秒 |

### 内存占用

- **小项目** (< 10k tokens): < 50MB
- **中项目** (10k-50k tokens): 50-200MB
- **大项目** (> 50k tokens): 200-500MB

## 🎯 最佳实践

### ✅ 推荐

1. **使用文件类型过滤** - 减少不必要的文件
2. **设置 Token 限制** - 避免生成过大的输出
3. **分模块处理** - 不要一次性上传整个项目
4. **先测试小项目** - 熟悉界面操作
5. **保存常用配置** - 记住常用的过滤规则

### ❌ 避免

1. ❌ 上传包含 node_modules 的目录
2. ❌ 上传整个大型项目
3. ❌ 不设置任何过滤
4. ❌ 忽略警告提示
5. ❌ 在慢速网络下使用（虽然是本地，但文件读取需要时间）

## 🚀 部署到服务器

如果你想让团队成员都能使用：

### 方法 1: GitHub Pages

```bash
# 1. 推送到 GitHub
git add web/
git commit -m "Add web interface"
git push

# 2. 在 GitHub 仓库设置中启用 Pages
# 3. 选择 main 分支的 /web 目录
# 4. 访问 https://yourusername.github.io/code-to-prompt/
```

### 方法 2: Vercel/Netlify

```bash
# 1. 连接 GitHub 仓库
# 2. 设置构建目录为 web/
# 3. 自动部署
```

### 方法 3: 自己的服务器

```bash
# 复制 web/ 目录到服务器
scp -r web/ user@server:/var/www/code-to-prompt/

# 配置 Nginx
server {
    listen 80;
    server_name code2prompt.yourdomain.com;
    root /var/www/code-to-prompt;
    index index.html;
}
```
